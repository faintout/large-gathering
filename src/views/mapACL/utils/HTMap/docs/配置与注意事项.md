# HTMap 配置与注意事项

本文档详细说明了HTMap地图防腐层的配置方法、使用注意事项和常见问题解决方案，为开发者和配置管理员提供完整的指导。

## 概述

HTMap 支持多种地图引擎，每种引擎都有对应的SDK配置。本文档涵盖了从基础配置到高级使用的各个方面，帮助您避免常见问题并正确配置地图环境。

## SDK配置说明

### 支持的引擎

#### 1. 腾讯地图 (Tencent Map)
- **引擎ID**: `tencent`
- **状态**: 已支持
- **SDK类型**: TMap JavaScript API

#### 2. 四维地图 (MineMap)
- **引擎ID**: `minemap`
- **状态**: 已支持
- **SDK类型**: 四维图新 JavaScript API

#### 3. Mapbox GL
- **引擎ID**: `mapbox`
- **状态**: 已支持
- **SDK类型**: Mapbox GL JS

#### 4. 高德地图 (AMap)
- **引擎ID**: `amap`
- **状态**: 计划中
- **SDK类型**: 高德地图 JavaScript API

#### 5. 百度地图 (BMap)
- **引擎ID**: `baidu`
- **状态**: 计划中
- **SDK类型**: 百度地图 JavaScript API

### 配置文件结构

#### sdkConfig.js

SDK配置文件位于 `src/utils/HTMap/config/sdkConfig.js`，包含以下内容：

```javascript
// 腾讯地图SDK配置
export const tencentSDKConfig = {
  js: 'https://map.qq.com/api/gljs?v=1.exp&key=YOUR_API_KEY',
  css: 'https://map.qq.com/api/gljs/css/1.exp.css',
  version: '1.exp'
}

// 四维地图SDK配置
export const minemapSDKConfig = {
  js: 'https://minedata.cn/minemapapi/v2.0.0/minemap.js',
  css: 'https://minedata.cn/minemapapi/v2.0.0/minemap.css',
  version: '2.0.0'
}

// Mapbox GL SDK配置
export const mapboxSDKConfig = {
  js: 'https://api.mapbox.com/mapbox-gl-js/v2.15.0/mapbox-gl.js',
  css: 'https://api.mapbox.com/mapbox-gl-js/v2.15.0/mapbox-gl.css',
  version: '2.15.0',
  accessToken: 'YOUR_ACCESS_TOKEN' // 访问令牌
}
```

### SDK加载管理

#### 自动加载

HTMap 会在需要时自动加载对应的SDK：

```javascript
import HTMap from '@/utils/HTMap'

// 创建地图实例时，会自动加载对应引擎的SDK
const map = new HTMap('map-container', {
  engine: 'mapbox', // 会自动加载Mapbox GL SDK
  center: [118.177726746, 39.630245209],
  zoom: 15
})

// 初始化地图
await map.init() // 如果SDK未加载，会先加载SDK
```

#### 手动加载

如果需要手动控制SDK加载时机：

```javascript
import { loadEngineSDK, isSDKLoaded } from '@/utils/HTMap/config/sdkConfig.js'

// 检查SDK是否已加载
if (!isSDKLoaded('mapbox')) {
  console.log('正在加载Mapbox GL SDK...')
  await loadEngineSDK('mapbox')
  console.log('Mapbox GL SDK加载完成')
}
```

#### 添加新的地图引擎

要添加新的地图引擎支持：

1. 在 `sdkConfig.js` 中添加SDK配置
2. 在 `engineConfig.js` 中添加引擎信息
3. 创建对应的适配器类
4. 在 `MapFactory.js` 中注册新引擎

```javascript
// 1. 添加SDK配置
export const newMapSDKConfig = {
  js: 'https://example.com/newmap.js',
  css: 'https://example.com/newmap.css',
  version: '1.0.0'
}

// 2. 添加引擎信息
export const engineExtConfig = {
  // ... 其他引擎
  newmap: { 
    id: 'newmap', 
    name: '新地图引擎', 
    supported: true, 
    status: 'active' 
  }
}
```

## 基础配置要求

### 1. 地图容器

- **容器ID**: 确保传入的容器ID在DOM中存在
- **容器尺寸**: 容器必须有明确的宽高，建议设置 `width` 和 `height` 样式
- **容器可见性**: 容器必须可见，避免在隐藏状态下初始化地图

```javascript
// 正确的容器设置
<div id="map-container" style="width: 100%; height: 400px;"></div>

// 错误的容器设置
<div id="map-container"></div> <!-- 没有尺寸 -->
<div id="map-container" style="display: none;"></div> <!-- 隐藏状态 -->
```

### 2. API密钥配置

- **腾讯地图**: 需要在 `sdkConfig.js` 中配置有效的API密钥
- **四维地图**: 需要配置有效的访问令牌
- **Mapbox GL**: 需要在 `sdkConfig.js` 中配置有效的accessToken

```javascript
// 确保API密钥配置正确
export const tencentSDKConfig = {
  js: 'https://map.qq.com/api/gljs?v=1.exp&key=YOUR_ACTUAL_KEY',
  // ...
}

export const mapboxSDKConfig = {
  // ...
  accessToken: 'YOUR_ACTUAL_ACCESS_TOKEN'
}
```

## 使用限制

### 1. 坐标系统

- **坐标格式**: 所有坐标必须使用 `[经度, 纬度]` 格式
- **经度范围**: -180 到 180 度
- **纬度范围**: -90 到 90 度
- **精度要求**: 建议使用6位小数精度

```javascript
// 正确的坐标格式
map.setCenter([118.177726746, 39.630245209])

// 错误的坐标格式
map.setCenter([39.630245209, 118.177726746]) // 纬度在前，经度在后
map.setCenter([200, 100]) // 超出有效范围
```

### 2. 参数范围限制

- **缩放级别**: 不同地图引擎支持的缩放范围可能不同
- **倾斜角度**: 通常支持 0-60 度
- **旋转角度**: 支持 0-360 度

```javascript
// 获取当前引擎支持的参数范围
const minZoom = map.getMinZoom()
const maxZoom = map.getMaxZoom()
const minPitch = map.getMinPitch()
const maxPitch = map.getMaxPitch()

// 在有效范围内设置参数
map.setZoom(Math.max(minZoom, Math.min(maxZoom, 16)))
map.setPitch(Math.max(minPitch, Math.min(maxPitch, 30)))
```

### 3. 引擎差异

- **功能支持**: 不同地图引擎对某些功能的支持可能不同
- **参数行为**: 相同参数在不同引擎中可能有不同的表现
- **样式差异**: 地图样式和视觉效果可能存在差异

## 初始化注意事项

### 1. 异步初始化

- **必须等待**: 地图初始化是异步操作，必须使用 `await` 或 `.then()`
- **错误处理**: 建议使用 try-catch 包装初始化代码
- **就绪回调**: 使用 `onMapReady` 确保地图完全就绪后再操作

```javascript
// 正确的初始化方式
try {
  const map = new HTMap('map-container', options)
  await map.init()
  
  map.onMapReady(() => {
    console.log('地图已就绪，可以开始操作')
  })
} catch (error) {
  console.error('地图初始化失败:', error)
}

// 错误的初始化方式
const map = new HTMap('map-container', options)
map.init() // 没有等待初始化完成
map.setCenter([118.18, 39.64]) // 可能失败
```

### 2. 容器状态

- **容器可见**: 确保容器在初始化时是可见的
- **尺寸稳定**: 避免在初始化过程中改变容器尺寸
- **父容器**: 确保父容器没有影响子容器的显示

## 性能优化建议

### 1. 资源管理

- **延迟加载**: HTMap采用延迟加载策略，只有在需要时才加载SDK
- **缓存机制**: 已加载的SDK会被缓存，避免重复加载
- **资源清理**: 在页面卸载前调用 `destroy()` 方法清理资源

```javascript
// 在组件销毁时清理资源
onUnmounted(() => {
  if (map) {
    map.destroy()
  }
})
```

### 2. 操作优化

- **批量操作**: 避免频繁调用地图方法，考虑批量更新
- **防抖处理**: 对于用户输入的操作，考虑使用防抖处理
- **事件优化**: 合理使用事件监听器，避免内存泄漏

```javascript
// 使用防抖处理用户输入
import { debounce } from 'lodash-es'

const debouncedSetCenter = debounce((lng, lat) => {
  map.setCenter([lng, lat])
}, 300)

// 用户输入时调用
input.addEventListener('input', (e) => {
  debouncedSetCenter(e.target.value)
})
```

## 错误处理

### 1. 参数验证错误

- **格式错误**: 坐标格式不正确时会抛出错误
- **范围错误**: 参数超出有效范围时会抛出错误
- **类型错误**: 参数类型不正确时会抛出错误

```javascript
try {
  map.setCenter('invalid-format')
} catch (error) {
  if (error.message.includes('格式错误')) {
    console.error('坐标格式不正确')
  } else if (error.message.includes('超出范围')) {
    console.error('坐标超出有效范围')
  }
}
```

### 2. 网络错误

- **SDK加载失败**: 网络问题导致SDK无法加载
- **超时错误**: SDK加载超时
- **API调用失败**: 地图服务API调用失败

```javascript
try {
  await map.init()
} catch (error) {
  if (error.message.includes('timeout')) {
    console.error('SDK加载超时，请检查网络连接')
  } else if (error.message.includes('network')) {
    console.error('网络连接失败，请检查网络设置')
  } else {
    console.error('地图初始化失败:', error.message)
  }
}
```

### 3. 加载超时

SDK加载支持超时处理，默认超时时间为30秒：

```javascript
// 加载SDK时如果超时，会抛出错误
try {
  await loadEngineSDK('mapbox')
} catch (error) {
  if (error.message.includes('timeout')) {
    console.error('SDK加载超时，请检查网络连接')
  } else {
    console.error('SDK加载失败:', error.message)
  }
}
```

## 常见问题解决

### 1. 地图不显示

**可能原因:**
- 容器尺寸为0
- 容器被隐藏
- SDK加载失败
- API密钥无效

**解决方案:**
```javascript
// 检查容器状态
const container = document.getElementById('map-container')
console.log('容器尺寸:', container.offsetWidth, container.offsetHeight)
console.log('容器可见性:', container.style.display, container.style.visibility)

// 检查SDK状态
import { isSDKLoaded } from '@/utils/HTMap/config/sdkConfig.js'
console.log('SDK已加载:', isSDKLoaded('mapbox'))
```

### 2. 地图操作无响应

**可能原因:**
- 地图未初始化完成
- 参数格式错误
- 方法不存在

**解决方案:**
```javascript
// 检查地图状态
if (map.mapInstance) {
  console.log('地图已初始化')
} else {
  console.log('地图未初始化')
}

// 检查方法是否存在
if (typeof map.setCenter === 'function') {
  console.log('setCenter方法存在')
} else {
  console.log('setCenter方法不存在')
}
```

### 3. 性能问题

**可能原因:**
- 频繁调用地图方法
- 大量标记点或图层
- 事件监听器过多

**解决方案:**
```javascript
// 使用节流处理频繁操作
import { throttle } from 'lodash-es'

const throttledSetZoom = throttle((zoom) => {
  map.setZoom(zoom)
}, 100)

// 清理不需要的事件监听器
map.off('click', clickHandler)
```

## 常见问题

### Q: 如何更换地图引擎的API密钥？
A: 修改 `sdkConfig.js` 中对应引擎的配置，更新API密钥或访问令牌。

### Q: 如何添加自定义的地图引擎？
A: 按照"添加新的地图引擎"部分的步骤，创建新的适配器和配置。

### Q: SDK加载失败怎么办？
A: 检查网络连接、API密钥配置，以及SDK文件地址是否正确。

### Q: 如何优化SDK加载性能？
A: 使用CDN加速、启用HTTP/2、考虑预加载关键SDK等。
