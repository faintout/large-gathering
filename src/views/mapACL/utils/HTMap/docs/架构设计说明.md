# HTMap 架构设计说明

本文档详细说明了HTMap地图防腐层的架构设计、设计原则、实现优势和当前代码架构，为开发团队提供清晰的架构指导。

## 概述

HTMap 是一个地图防腐层框架，旨在为多种地图引擎提供统一的 API 接口。本文档详细说明了 HTMap 的架构设计、设计原则和实现优势。

### 核心组件说明

#### 1. HTMap 主入口 (`src/utils/HTMap/index.js`)
- **职责**: 地图实例管理、方法路由分发、统一接口封装
- **特点**: 不包含业务逻辑，专注于接口统一和路由分发
- **优势**: 简化了上层应用的调用方式

#### 2. BaseAdapter 基类 (`src/utils/HTMap/adapters/BaseAdapter.js`)
- **职责**: 参数验证、格式转换、通用功能实现
- **特点**: 抽象类，定义所有地图引擎必须实现的接口
- **优势**: 统一了参数验证逻辑，避免重复代码

#### 3. 具体适配器实现
- **TencentMap**: 腾讯地图引擎适配器
- **MapboxGL**: MapboxGL 引擎适配器  
- **MineMap**: 四维地图引擎适配器

#### 4. MapFactory 工厂类 (`src/utils/HTMap/core/MapFactory.js`)
- **职责**: 根据引擎类型创建对应的适配器实例
- **管理**: 管理适配器的创建逻辑
- **扩展**: 支持动态扩展新的地图引擎

## 设计原则

### 1. 单一职责原则 (Single Responsibility Principle)
- 每个组件都有明确的职责分工
- HTMap 主入口只负责路由分发
- BaseAdapter 只负责参数验证和通用功能
- 具体适配器只负责特定引擎的实现

### 2. 开闭原则 (Open-Closed Principle)
- 对扩展开放：可以轻松添加新的地图引擎适配器
- 对修改封闭：修改现有功能不需要改动核心架构

### 3. 依赖倒置原则 (Dependency Inversion Principle)
- 上层应用依赖抽象接口（HTMap）
- 具体实现依赖抽象基类（BaseAdapter）

### 4. 接口隔离原则 (Interface Segregation Principle)
- 每个适配器只实现必要的方法
- 避免实现不需要的接口方法

## 核心优势

### 1. **架构层次清晰**
- 分层明确，职责边界清晰
- 便于理解和维护代码结构
- 符合软件工程最佳实践

### 2. **避免重复代码**
- 参数验证逻辑在 BaseAdapter 中统一处理
- 所有适配器共享相同的验证规则
- 维护时只需要修改一个地方

### 3. **易于扩展和维护**
- 添加新地图引擎只需实现 BaseAdapter 的子类
- 修改验证逻辑只需修改 BaseAdapter
- 各层之间耦合度低，修改影响范围小

### 4. **性能优化**
- 避免了重复的参数验证
- 直接调用适配器方法，减少中间层开销
- 缓存机制和事件管理优化

### 5. **代码复用性高**
- 通用功能在基类中实现
- 验证逻辑可以被所有适配器复用
- 减少代码重复，提高开发效率

### 核心配置文件

#### 1. engineConfig.js - 引擎配置和状态管理
**主要功能：**
- 管理地图引擎的基础信息（id、name、supported、status）
- 提供引擎状态枚举（EngineStatus、EngineType）
- 提供引擎管理方法（getActiveEngines、isEngineSupported等）

#### 2. sdkConfig.js - SDK配置和资源加载管理
**主要功能：**
- 管理各地图引擎的SDK配置（JS、CSS文件地址、版本信息）
- 提供SDK加载方法（loadEngineSDK）
- 内置资源加载功能（loadCSS、loadJS、waitForGlobalVariables）
- 管理MapboxGL的accessToken等敏感配置

#### 3. defaultConfig.js - 智能配置管理
**主要功能：**
- 为各地图引擎生成通用配置
- 包含中心点、缩放级别、最大缩放等基础配置
- 包含所有引擎的特定配置（tencentConfig、mapboxConfig等）
- 通过智能过滤机制，确保只有相关引擎才能访问对应配置

## 实现示例

### 参数验证流程

```javascript
// 1. 应用层调用
state.mapObject.setCenter(controlValues.center)

// 2. HTMap 主入口路由
setCenter(center) {
  if (this.mapInstance && this.mapInstance.setCenter) {
    return this.mapInstance.setCenter(center)  // 直接调用，无验证
  }
  // ... 其他逻辑
}

// 3. BaseAdapter 参数验证
setCenter(center) {
  // 完整的参数验证逻辑
  let validCenter = this._validateCenter(center)
  // 调用子类实现
  this._setCenter(validCenter)
}

// 4. 具体适配器实现
_setCenter(center) {
  // 直接调用地图引擎 API
  this.map.setCenter(center)
}
```

### 扩展新引擎

```javascript
// 1. 继承 BaseAdapter
export default class NewMapAdapter extends BaseAdapter {
  constructor(containerId, options) {
    super(containerId, options)
  }
  
  // 2. 实现必要的方法
  _setCenter(center) {
    // 新引擎的具体实现
    this.map.setCenter(center)
  }
  
  // 3. 注册到 MapFactory
  // 自动获得所有验证逻辑和通用功能
}
```

## 使用流程

### 1. 创建地图实例
```javascript
const map = new HTMap('map-container', {
  engine: 'mapbox',
  center: [118.177726746, 39.630245209],
  zoom: 15
})
```

### 2. 初始化地图
```javascript
try {
  await map.init()
  
  map.onMapReady(() => {
    console.log('地图已就绪')
  })
} catch (error) {
  console.error('地图初始化失败:', error)
}
```

### 3. 使用地图功能
```javascript
// 设置地图视角
map.setViewport([118.18, 39.64], 16, {
  pitch: 30,
  bearing: 45
})

// 获取地图状态
const center = map.getCenter()
const zoom = map.getZoom()
```
